# Atelier Studio 1.3.5

Atelier Studio 1.3.5 is a feature and integration release focused on knowledge-grounded conversations, new agent providers, and a stronger scientific writing loop. It introduces a complete Knowledge Base surface with source capture, per-thread context pills, and citation-aware answers; adds Kimi as a native ACP provider and OpenCode over a shared Rust ACP client; makes the compiled-PDF tab of the LaTeX studio fully annotatable with SyncTeX round-tripping; and ships faster packaged startup, a reorganized rail, and sccache-backed build tooling.

## Highlights

- Added a full Knowledge Base surface: capture files, PDFs, web pages, notes, YouTube transcripts, Obsidian folders/vaults, and gbrain pages as reusable sources, attach them per conversation, and get answers with clickable citations that open the source at the cited location.
- Added Kimi as a native Rust ACP provider with dynamic models, thinking support, auth probes, permissions/images/sessions bridging, and Kimi K3 with a 1M-token context ring.
- Added OpenCode as a provider over a new shared Rust ACP client, with a shared process, real interruption, dynamic models, and favorites.
- The LaTeX studio's "PDF ↗" tab is now the full PDF viewer — persistent highlights, underlines, notes, and send-to-chat — while staying SyncTeX-linked to the editor: cursor moves mark the PDF, double-clicking the PDF jumps to the source line, and recompiles reload in place without losing annotations.
- Factored the selection pill (viewers, LaTeX studio, injected HTML reports) into one shared module with identical behavior everywhere.
- Made packaged startup interactive sooner, with native soak timestamps and a recorded baseline.
- Reorganized the rail: Browser and Terminal moved to the TopBar, an "Other surfaces" drawer keeps IDE and Gallery front and center, and a compact New chat button was added.
- Added sccache-backed Tauri build wrappers with per-worktree targets and cache status/prune commands.

## Knowledge Base

- Added a dedicated Knowledge surface (atelier tab) with a shared actions hook, collections, chips, collapsible groups, recents, sorting, and archive (plans 049–052).
- Added a source store with extraction for files, PDFs, web pages, and notes, plus the `atelier-kb` CLI (plan 049 T1) with an inter-process lock, corrupted-registry recovery, and PDF freshness checks.
- Added full-page browser capture with chunked transfer (WKWebView truncates titles at 1,000 characters), a 24k fallback and backend fetch, off-main-thread captures (a 9-second freeze previously blocked chunks), and browser-agnostic User-Agent with actionable errors on anti-bot sites.
- Added composer integration: a source picker, per-conversation pills persisted per thread, an aggregated pill from 3 sources up, "sent with N sources" metadata under the bubble, and a badge with title tooltips on the book button.
- Added `atelier-kb` block injection at send time with dual Node/Rust backend support and byte-for-byte parity, plus stripping of the block from native history reloads.
- Added citation-aware answers: readable titles instead of hexadecimal ids, `cite`/`location`/`markdownLink` per source type, click-to-open at the cited location, PDF citations opening in the right-hand panel at the cited page, and file citations routed to the project viewer or Preview.
- Added gbrain integration: corpus search, page pinning, inbox capture, and direct page writes with guardrails; gbrain resolved via `~/.bun/bin`.
- Added YouTube sources with timestamped transcripts and channel support, Obsidian folders and vaults as composed sources, and a Zotero-reader book button with citation deep links.
- Added registry v2 with collections and archive, multi-select with batch tag/archive/attach actions, and active-collection pinning.
- Made `kbList` native in Rust (~1 ms, no Node spawn) with a 30-second front-end cache TTL; the Knowledge panel is available from the home screen.
- Hardened the pipeline end to end: thread-correlated auto-attach, `kbRemove` purge, sanitized titles, staging of `kb_prompt.mjs` in the Rust server dist with an import guardrail, and disambiguated block headers ("my reference / my article" refers to these sources).

## Kimi provider (native ACP)

- Added a native Rust ACP provider for Kimi, with structured errors and asynchronous reverse RPC in the shared ACP layer, plus a deterministic ACP fixture for tests.
- Bridged permissions, configuration, images, and sessions; added Node backend parity and UI-parity/live-smoke test guards.
- Exposed Kimi authentication and dynamic models in setup — the auth probe also feeds the HTTP `/setup` endpoint — and the model catalog with thinking is available from boot instead of a lone "CLI default".
- Added Kimi K3 (`openrouter/moonshotai/kimi-k3`) to the chat with a 1M-token context ring and to the OpenCode catalog.
- Made `/name` skills work in Kimi chats.
- Fixed the store so Kimi is recognized by `known_provider` and threads no longer fall back to Claude.

## OpenCode provider (shared ACP client)

- Added a shared Rust ACP client and wired OpenCode over ACP (plan 045), with a wire contract probed against the real binary.
- The Node sidecar runs OpenCode on ACP with a shared process, real interruption, and common helpers.
- Added dynamic models and favorites in settings and the model picker, and OpenCode in the new-chat modal (2×2 grid).

## LaTeX studio, PDF, and gallery

- The "PDF ↗" button now opens the compiled PDF in the full PDF viewer — persistent highlights/underlines/strikethroughs/notes stored server-side, color swatches, and send-to-chat — connected to the editor tab over the existing SyncTeX channel: compile reloads the document in place (annotations preserved), the editor cursor drives a position marker in the PDF, and double-clicking the PDF jumps to the source line. Out-of-project files keep the previous SyncTeX-only tab.
- Factored the selection pill into a shared `sel_pill.js` module used by the PDF viewer, the LaTeX studio, and the overlay injected into project HTML reports: one implementation of the Studio "Add to chat" restyle, the Claude-session target picker, direct send, and keyboard handling. Each host keeps its own selection capture, payload, and placement. Also fixes the embedded report pill reverting to "↑" after a send.
- Added a rewrap-document button to the LaTeX studio toolbar.
- Project `.html` files now open as rendered pages instead of source (targeted line or diff views still open the code editor).
- Fixed chat file attachments routing.
- Reduced the annotation capsule send button to 30 px and made the capsule auto-grow for long text.

## Startup performance

- Made packaged startup interactive sooner and timestamped soak runs natively.
- Recorded a dated startup soak and completed the historical baseline (plans in `docs/plans/`).

## Interface

- Moved Browser and Terminal from the rail to the TopBar.
- Added an "Other surfaces" drawer — IDE and Gallery remain visible by default; Highlights lives in the drawer.
- Added a compact New chat button with a compose (pencil + square) icon.
- Regularized the UI overhaul: `RowButton` is now tracked by the CSS contract tests.

## Build and release engineering

- Added sccache-backed Tauri build wrappers: per-worktree `target/` directories, a 10 GiB shared compile cache, `rust:cache:status`, and preview-first `rust:targets:prune`.
- CI now stages native release resources and installs mobile dependencies for release runs.
- Runtime fix: the event pump is drained before the synthetic `done`, so late provider events are no longer lost.
- Workspace fix: the "N files modified" card no longer counts pre-existing untracked files.

## Upgrade notes

- Existing projects, conversations, gallery state, highlights, and provider settings remain compatible.
- Knowledge Base sources are stored locally per machine; review attached sources per conversation before sending sensitive threads.
- Kimi and OpenCode require their respective CLIs; both are probed at setup and configurable in settings.
- The Rust backend remains the default for chat and gallery. `ATELIER_BACKEND=node` and `ATELIER_GALLERY_BACKEND=node` are available only for explicit compatibility testing.
- Restart project-scoped gallery servers after changing gallery assets; the packaged app handles this during a normal upgrade.

## Platform and prerequisites

- macOS on Apple Silicon.
- Claude Code CLI for Claude sessions; Codex CLI for Codex sessions; Kimi and OpenCode CLIs for their providers.
- Node.js and the project gallery runtime are bundled with the desktop application.

If macOS Gatekeeper blocks the downloaded build, run:

```bash
xattr -cr "/Applications/Atelier.app"
```
